# 서비스 기획/디자인 가이드 (v1)

## 1) 설문지 배치
- 권장 온보딩 순서: `질문(설문) -> 사주 -> 타로 -> 결과`
- 의도:
  - 설문으로 개인화 신호를 먼저 수집해 사주/타로 해석 품질을 높임
  - 사용자가 초반에 "참여했다"는 감각을 가져 이탈률을 낮춤
- 설문 기본 원칙:
  - 3~7문항, 60~90초 내 완료
  - 선택형 중심 + 자유입력 1문항 이하
  - 결과와 직접 연결되는 항목만 수집

## 2) 보안 레벨 (초기 권장안)
- 접근 방식: `토큰 기반 인증(JWT) + 만료시간 + HTTPS`
- 최소 보안 요구사항:
  - 서버에서 토큰 서명/만료 검증 필수
  - 액세스 토큰 짧게 운영 (예: 15~60분)
  - 재발급 토큰은 안전 저장(HTTP-only 쿠키 권장)
  - 주요 API에 rate limit 적용
  - 민감 데이터(생년월일 등) 저장 시 암호화 적용
  - 로그에 개인정보 원문 기록 금지
- 운영 체크:
  - 키 로테이션 정책 수립
  - CORS 허용 도메인 최소화
  - 에러 메시지에 내부 정보 노출 금지

## 3) 타로 이미지 스타일 가이드
- 방향: `한국 전통 모티프`
- 핵심 키워드:
  - 단청 패턴, 한지 질감, 오방색 포인트, 전통 문양
  - 붓터치/목판화 느낌, 은은한 금박 디테일
- 톤앤무드:
  - 신비롭지만 과도하게 어둡지 않음
  - 상징은 분명하고 배경은 절제
- 금지 요소:
  - 서양 고딕만 강조된 클리셰 표현
  - 과도한 네온/사이버펑크 톤

## 4) 결과 캐릭터 이미지 스타일 가이드
- 방향: `일러스트 + 지역 분위기 강조`
- 구성 원칙:
  - 캐릭터(전경) + 지역성 요소(중경/배경) 분리
  - 지역별 색감/소품/건축/자연 요소를 2~3개 핵심으로 사용
  - 표정/포즈는 결과 성향 키워드와 일치시킴
- 예시 지역 반영:
  - 해안: 바람/파도/청록-모래색 팔레트
  - 산지: 안개/능선/녹청-회갈색 팔레트
  - 도심: 간판 리듬/콘크리트 톤 + 포인트 컬러

## 5) 생성 프롬프트 템플릿

### 5-1. 타로 카드 이미지
```text
[카드명]을 한국 전통 모티프로 재해석한 타로 일러스트.
단청 패턴, 한지 질감, 오방색 포인트, 전통 문양 배경.
상징은 선명하게, 배경은 절제된 구성.
신비롭지만 따뜻한 분위기, 고해상도 디지털 일러스트.
```

### 5-2. 결과 캐릭터 이미지
```text
[결과 성향]을 표현하는 캐릭터 일러스트.
[지역명]의 분위기를 반영한 배경(색감/소품/건축 요소 포함).
캐릭터는 전경에서 감정과 성향이 보이도록, 배경은 지역 정체성을 명확히.
과장된 포토리얼리즘 대신 감성적 일러스트 스타일.
```

## 6) 화면/UX 적용 기준
- 온보딩 진행 표시: `질문 1/4 -> 사주 2/4 -> 타로 3/4 -> 결과 4/4`
- 단계 이탈 방지:
  - 각 단계 상단에 소요 시간 안내
  - 설문 완료 시 즉시 다음 단계 진입
- 결과 화면:
  - 해석 텍스트 + 캐릭터 이미지 + 타로 요약을 한 화면에 배치
  - 저장/공유 버튼은 결과 하단 고정
